AI Story Generator

Problem: Move past an LLM-API wrapper: ship a streaming, RAG-grounded, agentic AI feature that stays reliable when the provider fails.

Stack: Vercel AI SDK, RAG over pgvector (Neon), TypeScript, Zod, Vitest, Playwright, GitHub Actions.

Outcome: An AI-engineering flagship: token streaming, retrieval grounding, a tool-calling agent, typed outputs, guardrails, evals, and observability.

  • AI SDK + streaming — streamText with token streaming; Groq (llama-3.3-70b-versatile) via OpenAI-compatible API, swappable through AI_BASE_URL/AI_API_KEY.
  • RAG + agent — pgvector retrieval on Neon grounds a bounded tool-calling loop; Zod-typed outputs and prompt-injection guardrails.
  • Evals + observability — golden-set graders (`npm run eval`); per-generation telemetry (tokens, latency, tool calls), plus a deterministic local fallback.
  • CI + E2E — typecheck, Vitest, node tests, and Playwright; GitHub Actions with required checks and Vercel previews.

Impact: Streaming AI with retrieval grounding and typed outputs—users still get a story when the provider fails.

Verify CI locally

Every featured repo ships a GitHub Actions workflow (lint, tests, build, and Playwright E2E) that runs on every push and pull request. You can also run the exact same pipeline locally with the steps below.

  1. Clone a featured repo, then npm ci.
  2. Run npm run ci for the core pipeline (same steps as CI minus E2E).
  3. Run npx playwright install chromium once, then npm run ci:full to include E2E—or npm run test:e2e after npm run ci.
  • mini-ecommerce — lint, type-check, Vitest, production build
  • mini-job-board — JS syntax, JSON schema validation, filter unit tests
  • ia-stories — typecheck, Vitest (64 tests), .env.example validation; E2E in ci:full

Workflow files live in .github/workflows/ci.yml in each repo; READMEs repeat these commands for reviewers.
